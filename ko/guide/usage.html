사용 설명서 ​

이 설명서는 관리자로서 관리자 패널을 사용하는 방법과 사용자가 텔레그램 봇과 상호 작용하는 방법을 안내합니다.

봇 명령 (사용자 측) ​

텔레그램 봇과 상호 작용하는 사용자를 위한 일반적인 명령:

  • /start - 봇 시작, 메인 메뉴 보기
  • 에너지 구매 - 메뉴 버튼 클릭, 기간 및 금액 선택
  • 주문 확인 - 최근 임대 주문 상태 조회
  • 사용자 센터 - 잔액, 추천 링크 등 보기
  • 고객 지원 문의 - 고객 서비스 연락처 정보 얻기

관리자 패널 (관리자 측) ​

관리자 패널 주소에 액세스하고 로그인하면 다음 모듈을 볼 수 있습니다:

1. 대시보드 ​

오늘의 주문 수, 총 매출액, 신규 사용자 등을 포함한 시스템 개요 데이터를 봅니다.

2. 주문 관리 ​

  • 에너지 주문: 모든 사용자의 에너지 임대 기록을 보고 상태(성공, 실패, 진행 중)별로 필터링을 지원합니다.
  • 주문 보상: 실패한 주문의 경우 수동으로 보상하거나 환불을 시도합니다.

3. 사용자 관리 ​

  • 봇을 사용한 모든 사용자 목록을 봅니다.
  • 사용자 잔액 수정(충전/차감).
  • 위반 사용자 차단.

4. 봇 관리 ​

  • 봇 다시 시작: 구성을 수정한 후 변경 사항을 적용하려면 이 버튼을 클릭하십시오.
  • 푸시 메시지: 모든 사용자 또는 특정 사용자에게 메시지를 방송합니다.

5. 에너지 풀 통합 ​

  • 에너지 풀 계정 생성: 외부 호출을 위한 API 키 생성.
  • 타사 풀 통합: 업스트림 에너지 풀 API 정보를 구성합니다.

자주 묻는 질문 (FAQ) ​

Q: 봇이 응답하지 않으면 어떻게 합니까? ​

A:

  1. Docker 컨테이너가 정상적으로 실행되고 있는지 확인합니다(docker compose ps).
  2. bot 컨테이너 로그(docker compose logs -f bot)에 오류가 있는지 확인합니다.
  3. 토큰이 올바른지, 서버 네트워크가 텔레그램 API에 연결할 수 있는지 확인합니다.

Q: 에너지를 받지 못했습니까? ​

A:

  1. 지급 주소에 거래 수수료에 충분한 TRX가 있는지 확인합니다(필요한 경우).
  2. 에너지 풀에 충분한 유휴 에너지가 있는지 확인합니다.
  3. 관리자 패널의 주문 세부 정보에서 오류 메시지를 봅니다.

유지 관리 조언

데이터 손실을 방지하기 위해 데이터베이스(nl-admin.sql 또는 /var/lib/mysql 디렉토리)를 정기적으로 백업하십시오.
